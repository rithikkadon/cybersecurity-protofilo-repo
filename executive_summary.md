This project presents a basic penetration testing assessment conducted in a controlled lab environment using platforms such as TryHackMe and DVWA. The objective of this project was to identify common web application vulnerabilities and understand how attackers exploit them.

Various security tools including Burp Suite and Hydra were used to perform testing activities such as intercepting HTTP requests, modifying parameters, and conducting password cracking attempts.

The assessment identified several vulnerabilities including SQL Injection, Cross-Site Scripting (XSS), and weak authentication mechanisms. These vulnerabilities can lead to unauthorized access, data leakage, and compromise of user accounts if not properly mitigated.

The results highlight the importance of secure coding practices, proper input validation, and strong authentication mechanisms. Recommendations such as implementing parameterized queries, enforcing strong password policies, and using security headers have been suggested.
